# 【不是好评大师】QQ 群锐评机器人

通过 NapCatQQ 监听 QQ 群消息，@机器人 + 关键词触发大模型生成回复。

## 快速开始

1. 复制 config_example.yaml 为 config.yaml，然后根据你的配置修改：

   cp config_example.yaml config.yaml

2. 编辑 config.yaml：

   napcat:
     http_api: NapCatQQ HTTP API 地址（默认 http://127.0.0.1:3000）
     access_token: NapCatQQ 访问令牌（在 NapCatQQ WebUI 中设置）

   bot:
     qq: 你的机器人 QQ 号
     allow_groups: 允许响应的群号（多个用逗号分隔）

   llm:
     api_key: 你的大模型 API Key
     api_base: 大模型接口地址
     model_name: 模型名称

3. 代码直接运行：

   Windows：双击 start.bat
   Linux：  ./start.sh

   打包为可执行文件：

   Windows：双击 build.bat
   Linux：  ./build.sh

## 环境要求

- Go 1.25+
- NapCatQQ 本地运行
- OpenAI 兼容的大模型 API（DeepSeek、豆包、通义等）

## 项目结构

  main.go           - 入口
  config/           - 配置加载
  cache/            - 消息环形缓存
  llm/              - 大模型客户端
  onebot/           - NapCatQQ HTTP API
  bot/              - 消息过滤 + 轮询
  cmd/              - 命令路由 + handler

## 扩展新命令

1. 在 config.yaml 的 cmd: 下添加配置
2. 在 cmd/ 下新建 handler 文件
3. 在 cmd/router.go 添加路由

## 部署说明

- 本机或云服务器均可，无需公网 IP
- 纯内网通信：Go 后端通过 HTTP 轮询 NapCatQQ 本地 API
- 编译为单二进制文件，零运行时依赖
